## Proposed Changes

### 1. Multi-Agent Data Preprocessing Pipeline

We will completely revamp the existing agents to fit a dynamic code-generation workflow:

#### [MODIFY] `agents/inspector.py`
- Analyzes the data for missing values and anomalies.
- Generates a detailed statistical report that serves as the prompt for the next agent.

#### [MODIFY] `agents/coder.py` (and potentially `planner.py`)
- Instead of using hard-coded dictionary mapping, the Coder agent will use an LLM.
- **Goal:** The LLM will write actual Python code using Pandas and Scikit-Learn to apply the correct preprocessing (e.g., `StandardScaler`, `OrdinalEncoder`, imputations) based on the Inspector's report.
- The output of this agent will be executable Python source code.

#### [MODIFY] `agents/reviewer.py`
- The Reviewer agent will act as a validator.
- **Goal:** It will safely execute the Python code generated by the Coder.
- It will verify the output shape, check if there are any remaining missing values or errors, and return feedback. If it fails, LangGraph can loop back to the Coder for a fix (optional future feature).

#### [MODIFY] `agents/graph_pipeline.py`
- Update the LangGraph state to pass the generated `python_code` and `execution_results` between nodes.

---

### 2. Robust Testing (Pytest)

#### [NEW] `agents/conftest.py`
We will create a Pytest fixture that returns a small, messy dummy DataFrame so tests run fast without loading the large `electriccard.csv`.

#### [MODIFY] `agents/test_inspector.py`, `test_coder.py`, `test_reviewer.py`, `test_pipeline.py`
- Convert existing script files into proper Pytest tests.
- Add strict assertions to ensure each agent functions correctly in isolation and as a pipeline.

## Verification Plan

### Automated Tests
- Run `pytest agents/` to verify that all agents pass their assertions.

### Manual Verification
- Run `python agents/graph_pipeline.py` and inspect the generated Python code printed by the Coder, as well as the final verification report printed by the Reviewer.
